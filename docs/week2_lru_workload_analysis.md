# 1. 实验目的

本实验旨在观察 LRU 在不同访问模式和不同缓存容量下的命中率变化，验证 workload 特征对缓存行为的影响，并为后续 throughput 与总耗时分析提供基础。

# 2. 实验配置

- 缓存策略：LRU
- Workload：Uniform / Hotspot-like
- Capacity：50 / 100 / 200 / 500
- 指标：命中率
- 可复现性：固定 seed，固定 key_space

# 3. Workload 说明

- **Uniform**：均匀随机访问，局部性较弱
- **Hotspot-like**：热点访问模式，约 80% 请求落在前 20% key 空间，局部性更强

选择这两类 workload，是为了对比“弱局部性”和“强局部性”场景下 LRU 的表现差异。

# 4. 命中率结果

| Capacity | Uniform | Hotspot-like |
| -------- | ------- | ------------ |
| 50       | 4.83%   | 17.69%       |
| 100      | 9.63%   | 34.54%       |
| 200      | 19.65%  | 62.76%       |
| 500      | 48.27%  | 86.93%       |

从结果可以看出，随着缓存容量增大，LRU 命中率持续上升；同时 Hotspot-like 负载下的命中率始终明显高于 Uniform。

# 5. 初步结论

1. 容量增大时，LRU 缓存命中率持续提升。
2. Hotspot-like 下命中率显著高于 Uniform，说明 LRU 对时间局部性更强的访问模式更敏感。
3. 当缓存容量逐渐接近热点集合大小时，命中率提升更加明显。
4. 这是因为热点 key 在 Hotspot-like 场景下会被重复访问，更容易长期保留在缓存中；而 Uniform 场景下访问更分散，缓存替换更频繁。
5. 当前实验主要关注命中率，后续还需要补充 throughput 和总耗时统计，以增强实验结论的完整性。
